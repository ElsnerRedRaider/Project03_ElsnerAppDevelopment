# Project 03 Scope Statement

**Project Name:** Elsner App Development Services — Website Prototype & Project Package

---

## Project Purpose
**Why are we doing this project?**

This serves two purposes at once. For CMPA3301, it's my final project — a complete, documented project package (planning docs, a working two-page website, and a retrospective) in a single public GitHub repo, showing I can manage a digital project start to finish.

It's also the first real step toward Elsner App Development Services, a business I intend to actually build, using HomeProof as its flagship proof of concept. I'm scoping this as a small, honest prototype of that future business's web presence, not a finished production site. Keeping it tight means I finish something I can be proud of and build on later, instead of overcommitting on a course timeline.

## Description
**How would you describe this project?**

A single public GitHub repository containing:
* A README.md that explains the project and links to the live site and documentation
* A docs/ folder with three documents: scope.md, plan.md, retrospective.md
* A two-page website (index.html, about.html) styled with one shared style.css, deployed live on GitHub Pages

The site introduces Elsner App Development Services and uses HomeProof as the concrete example of the kind of work the business does.

## Desired Results
**What specific outcomes must we achieve to be successful?**

* Public GitHub repo with a clean, professional structure
* README.md that clearly explains the project and links to the live site and docs/
* docs/scope.md — this document, revised after Assignment 04 peer feedback
* docs/plan.md — project plan with a TAME risk analysis and a WBS task schedule
* docs/retrospective.md — reflection on what went well, what went wrong, what I'd do differently
* index.html — a well-structured homepage introducing Elsner App Development Services
* about.html — a second page, linked from the homepage, with more background and the HomeProof example
* style.css — one stylesheet applied consistently across both pages
* Site live and viewable on GitHub Pages
* Meets the CMPA3301 final project rubric

## Exclusions
**What is out of the project's scope and should NOT be included?**

* No services page, contact form, or pages beyond index.html and about.html — this is a two-page prototype by design
* No live or embedded version of the HomeProof app — description and/or static images only
* No backend, database, CMS, or form submission functionality
* No SEO, analytics, or marketing integrations
* No business registration, licensing, contracts, or client-facing promises — Elsner App Development Services stays a concept for this project, not an operating business
* No claims of a client history, team, or track record I don't actually have
* Building this out into a full production site is a future phase, not part of this project
